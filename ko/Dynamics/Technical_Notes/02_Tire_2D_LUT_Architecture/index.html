    2D LUT 기반 타이어 동역학 모델 아키텍처 (Combined Slip)

    1. 개요 (Overview)

    기존의 해석적(Analytic) Magic Formula 타이어 모델은 횡방향(코너링)과 종방향(가감속) 중 하나만 독립적으로 계산하는 한계가 있었습니다. 이를 극복하고 극한의 한계 주행(Limit Handling)을 NMPC로 완벽히 제어하기 위해, 마찰 타원(Friction Ellipse) 물리를 반영한 2D Look-Up Table (LUT) 아키텍처를 새롭게 설계 및 도입했습니다.

    2. 물리적 직관: 타이어 그립의 한계 (Friction Ellipse)

    타이어가 노면과 비빌 수 있는 마찰력(그립)의 총합은 원형(타원) 형태로 한정되어 있습니다. 이를 '그립 예산'이라고 비유할 수 있습니다. * 타행 주행 (kappa = 0): 가속/감속 페달을 밟지 않으면 타이어는 가진 마찰력 100%를 코너링(횡력)에 쏟을 수 있습니다. 이때 횡력(\(F_y\))은 최대치를 기록하며 3D 지형도에서 가장 높은 산 능선을 형성합니다. * 급제동 / 급가속 (kappa \neq 0): 제동에 타이어 마찰력을 소모해 버리면, 코너링에 쓸 수 있는 마찰력은 급격히 줄어듭니다. 핸들을 꺾어도(alpha) 코너링 포스가 발생하지 않아 차가 바깥으로 밀려납니다 (언더스티어). 3D 지형도에서는 산의 높이가 푹 주저앉는 내리막길로 표현됩니다.

    2D LUT는 이 현상을 kappa(종방향 슬립)와 alpha(횡방향 조향각)라는 두 가지 입력을 받아, 해당 상황에서 타이어가 낼 수 있는 종/횡방향 힘을 거대한 3차원 지형도로 맵핑해둔 데이터베이스입니다. NMPC는 이 지형도를 미리 읽고 "제동을 너무 세게 하면 코너링 그립이 사라진다"는 것을 인지하여, 한계 주행 시 스스로 트레일 브레이킹(Trail Braking)과 같은 고도의 운전 기술을 구사하게 됩니다.

    3. 핵심 알고리즘 및 최적화

    3.1. 무차원 정규화 (Normalized LUT) 및 동적 스케일링

    특정 수직 하중(\(F_z\))에 종속된 타이어 힘(N)을 그대로 LUT에 고정해버리면, 차량이 급코너링을 할 때 발생하는 하중 이동(Weight Transfer) 현상을 제어기가 인지할 수 없습니다. 이는 극한 주행 시 스핀의 주된 원인이 됩니다. 이를 해결하기 위해 LUT에는 마찰 계수(\(\mu_x, \mu_y\)) 형태의 무차원(Normalized) 데이터만 저장합니다. 실제 타이어 힘은 NMPC 런타임 중에 현재 바퀴의 실시간 수직 하중 \(F_z\)를 곱하여 동적으로 팽창/수축시킵니다 (Dynamic Scaling).

    \(F_{y} = \mu_{y}(\kappa, \alpha) \times F_{z, \text{current}}\)

    3.2. AD 호환 \(C^1\) 연속성 (Catmull-Rom Bicubic Interpolation)

    Acados, CasADi와 같은 비선형 최적화 솔버의 내부 자동 미분(Auto-Differentiation, AD) 엔진은 함수의 미분값이 튀거나 불연속적인 꺾인 점(Kink)을 만나면 그래디언트가 폭발하여 연산을 실패합니다. 단순한 2D Bilinear Interpolation(\(C^0\) 연속)은 격자점 사이는 직선으로 이어지나 격자점에서 기울기가 꺾이는 치명적인 문제가 있습니다. 이를 방지하기 위해 Catmull-Rom Spline 곡선 기반의 Bicubic Interpolation을 C++ 템플릿 제네릭 프로그래밍(template <typename T>)으로 구현하였습니다. 그 결과 격자점을 매끄럽게 잇는 완벽한 \(C^1\) 곡면이 생성되어, 솔버의 연쇄 법칙(Chain Rule) 추적이 끊기지 않도록 방어막을 구축했습니다.

    4. 데이터 추출 및 튜닝 워크플로우

    사내 AI 타이어 분석 툴이나 차량 시뮬레이터(CarMaker, IPG 등)에서 촘촘한 해상도(예: 21x21 이상)로 정규화된 \(\mu_x, \mu_y\) 데이터를 추출합니다. 추출한 데이터를 CombinedSlipTireLUT.hpp 내부의 C++ 2차원 배열 포맷으로 덮어씌우고 다시 빌드하는 것만으로, NMPC 제어기가 새로운 타이어의 마찰 특성과 한계를 즉시 깨닫게 됩니다.
