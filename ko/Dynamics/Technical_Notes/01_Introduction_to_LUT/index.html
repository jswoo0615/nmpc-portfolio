    룩업 테이블(LUT)의 이해와 기초 구현

    1. 룩업 테이블(LUT)이란 무엇인가?

    룩업 테이블(Look-Up Table, LUT)은 복잡하고 무거운 수학적 연산(사인, 지수, 다항식 등)을 실시간으로 매번 계산하는 대신, 발생할 수 있는 입력값들에 대한 결과값을 미리 계산하여 메모리(배열)에 저장해 두고, 필요할 때 '찾아서(Look-up)' 꺼내 쓰는 최적화 기법입니다.

    • 핵심 철학: "계산 시간을 메모리 용량과 맞바꾼다 (Compute-to-Memory Trade-off)"
    • 장점: 수십 번의 복잡한 물리 수식 연산을 단 한 번의 메모리 참조(메모리 주소 접근)로 끝낼 수 있어 실시간(Real-Time) 제어기의 속도가 획기적으로 빨라집니다.
    • 단점: 배열(Grid)에 정확히 일치하지 않는 소수점 단위의 입력이 들어왔을 때는, 주변에 있는 배열 값들을 조합해 사이의 값을 유추하는 보간법(Interpolation)이 필수적입니다.

    2. 1D LUT vs 2D LUT

    • 1D LUT (\(y = f(x)\)): 입력이 1개인 1차원 배열(선)입니다. (예: alpha에 따른 Fy)
    • 2D LUT (\(z = f(x, y)\)): 입력이 2개인 2차원 배열(면/표)입니다. (예: kappa와 alpha에 따른 Fy). 엑셀의 표나 등고선 지도를 상상하시면 됩니다.

    3. 예시 코드: C++ 2D LUT와 쌍선형 보간 (Bilinear Interpolation)

    NMPC 코어 내부에서는 미분 폭발을 막기 위해 16개의 점을 사용하는 쌍3차 보간(Bicubic)을 사용하지만, 원리를 완벽하게 익히기 위해서는 가장 가까운 4개의 점을 사용하는 쌍선형 보간(Bilinear)부터 이해하는 것이 좋습니다.

    다음은 2D LUT의 값을 참조하고, 그 사이의 값을 보간하는 직관적인 C++ 예제입니다.

    #include <iostream>
#include <vector>
#include <cmath>

class Simple2DLUT {
private:
    // 3x3 2D LUT 데이터베이스 (예: X=조향각, Y=브레이크량)
    std::vector<std::vector<double>> table = {
        { 10.0, 20.0, 30.0 },
        { 15.0, 25.0, 35.0 },
        { 20.0, 30.0, 40.0 }
    };

    double x_min = 0.0, x_max = 2.0; // X축 범위
    double y_min = 0.0, y_max = 2.0; // Y축 범위
    double dx = 1.0, dy = 1.0;       // 격자 간격

public:
    double getValue(double x, double y) {
        // 1. 입력값이 테이블 밖으로 나가지 않게 제한 (Clamping)
        x = std::max(x_min, std::min(x, x_max));
        y = std::max(y_min, std::min(y, y_max));

        // 2. 현재 내 위치가 속한 격자의 인덱스(좌표) 찾기
        int x_idx = static_cast<int>((x - x_min) / dx);
        int y_idx = static_cast<int>((y - y_min) / dy);

        // 경계 조건 처리
        if (x_idx >= 2) x_idx = 1;
        if (y_idx >= 2) y_idx = 1;

        // 3. 내 위치를 둘러싼 가장 가까운 4개의 점(텐트의 기둥 4개) 값 읽어오기
        double V00 = table[y_idx][x_idx];         // 좌측 하단
        double V10 = table[y_idx][x_idx + 1];     // 우측 하단
        double V01 = table[y_idx + 1][x_idx];     // 좌측 상단
        double V11 = table[y_idx + 1][x_idx + 1]; // 우측 상단

        // 4. 격자 내부에서의 상대적 위치 (0.0 ~ 1.0) 계산 (비율)
        double tx = (x - (x_min + x_idx * dx)) / dx;
        double ty = (y - (y_min + y_idx * dy)) / dy;

        // 5. 쌍선형 보간 (Bilinear Interpolation) 적용
        // 먼저 X축 방향으로 아랫변과 윗변을 각각 선형 보간 (1D 보간 2번)
        double bottom = V00 * (1 - tx) + V10 * tx;
        double top    = V01 * (1 - tx) + V11 * tx;

        // 구해진 윗변과 아랫변을 Y축 방향으로 한 번 더 선형 보간
        return bottom * (1 - ty) + top * ty;
    }
};

int main() {
    Simple2DLUT lut;

    // 격자 정중앙 값 (x=0, y=0) -> V00 값인 10.0 출력
    std::cout << "LUT(0, 0) = " << lut.getValue(0.0, 0.0) << std::endl;

    // 격자 사이의 중간 값 (x=0.5, y=0.5) 보간 -> 4개 점의 평균값 출력
    std::cout << "LUT(0.5, 0.5) = " << lut.getValue(0.5, 0.5) << std::endl;

    return 0;
}

    코드 분석 요약

    1. 메모리 접근: 실시간 복잡한 수식 없이 2차원 배열 table[y][x]에서 주변의 4개 숫자만 빠르게 쏙 빼옵니다 (V00, V10, V01, V11).
    2. 비례식 계산(보간): 내가 두 격자의 한가운데(0.5)에 있다면, 양쪽 격자의 값을 정확히 50%씩 섞어서 반환합니다.

    대표님께서는 이 단순한 보간법의 한계(기울기가 꺾임)를 극복하기 위해, NMPC 코드 내에서 V00~V11 4개가 아닌 16개의 점을 끌어와 매끄러운 다항식 곡선을 그리는(Bicubic) 방식으로 업그레이드 하셨다고 이해하시면 완벽합니다!
