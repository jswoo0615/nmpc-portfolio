    NMPC ↔ Autoware (CARLA) 연동 가이드 (2026-07-19)

    우리 NMPC 컨트롤러를 Autoware 전체 스택(autoware_carla_interface + CARLA) 안에 꽂아 CARLA 차량을 운전시키는 절차. 오늘 달성한 것과 남은 벽, 다음 세션에서 빠르게 완료하는 법을 기록.

    달성 (2026-07-19)

    • 우리 NMPC 컨트롤러: Autoware autoware_control_msgs/Control 인터페이스로 포팅, ControlCore 공유(Jetson/CARLA 노드와 동일), 게이트 통과용 gear/turn/hazard 발행.
    • 전체 CARLA→Autoware→control 파이프라인 구동 확인, Autoware 베이스라인 컨트롤러가 CARLA 차량을 실제 완주(세션 앞부분, 첫 route).
    • 우리 컨트롤러 스왑 절차 완성: 컨트롤러 composable-node unload + 우리 노드를 /control/trajectory_follower/control_cmd로 발행.

    환경 요건 (이미 갖춤)

    • CARLA 0.9.15, carla_ros_bridge/carla_msgs, autoware_carla_interface(빌드됨)
    • Town10HD 지도: ~/autoware_data/maps/Town10HD/ (lanelet2 + pcd + map_projector_info.yaml)
    • 심링크: install/.../autoware_carla_interface.launch.xml → launch/... (경로 불일치 수정)

    넘은 벽 (핵심 함정들)

    1. VRAM OOM (RTX 4060 8GB): CARLA ~4GB 상주 + 전체 perception CUDA = OOM (pointcloud_container cudaErrorMemoryAllocation로 사망). → perception:=false.
    2. planning이 perception 입력 대기: perception:=false 시 behavior planning이 objects/occupancy_grid/pointcloud를 기다리며 멈춤. → dummy_perception.py로 빈 값 발행.
    3. sim-time 불일치: 시스템은 use_sim_time. 더미가 벽시계 stamp 쓰면 TF 변환 실패 ("extrapolation into the future"). → 모든 보조 노드에 -p use_sim_time:=true.
    4. motion planning(path→trajectory) 궤적 공백: behavior/path는 나오나 최종 trajectory 미발행 (planning_simulator에서도 동일). → path_to_trajectory 브리지로 우회.
    5. CARLA sync 세션 취약: 스택 구동 중 CARLA에 제3의 클라이언트 접속 금지 (ego 소멸/스트림 세션 붕괴). 디버깅은 ROS 토픽으로만.

    남은 벽 — 라우팅 (다음 세션 최우선)

    • 증상: route_handler: Failed to find a proper route / SetRoutePoints "route is empty". ego는 lanelet 위에 있고(3790, 0.2m) python lanelet2 getRoute는 OK인데 Autoware는 실패.
    • 근본 원인: CARLA autoware-contents Town10HD 지도가 format_version: null (lanelet2_map_loader: has no format_version). Autoware 라우팅 그래프 구성이 불안정. 첫 route 성공은 일시적.
    • 권장 수정 (다음 세션):
    • 지도에 유효 format_version 부여: lanelet2_map.osm에 MetaInfo/format_version=1.x.x 추가 (또는 정제된 CARLA lanelet2 맵 사용 — traffic light 포함판, README 참고).
    • 고정 스폰: autoware_carla_interface launch의 spawn_point:=[x,y,z,roll,pitch,yaw]로 연결성 좋은 lanelet 위에 스폰(랜덤 스폰 복불복 제거).
    • 도달 가능 목표 자동 계산: scripts/find_routable_goal.py가 ego lanelet에서 법적 진행방향 successor 체인을 따라 목표 좌표를 산출. scripts/set_route.py로 발행.

    실행 절차 (라우팅 해결 후)

    S=.../nmpc_ros2/launch/autoware_planning_sim.sh
# 터미널1: 전체 스택 (perception:=false)
bash "$S" carla_full 2>&1 | tee /tmp/e2e_log.txt
# 터미널2: 보조 도구 (dummy perception + path->trajectory 브리지, sim_time)
bash "$S" support
# RViz: 목표 1개 (재경로 금지 — mission_planner SIGSEGV). route 생성 확인.
# 터미널3: Autoware 컨트롤러 언로드
bash "$S" stop_controller
# 터미널4: 우리 NMPC
bash "$S" nmpc
# RViz: Engage(AUTO) → 우리 컨트롤러가 CARLA 차량 운전

    알려진 이슈

    • 재경로(완주 후 새 목표): mission_planner_container SIGSEGV. 스택당 목표 1개.
    • pose_instability_detector 등 일부 진단 노드 abort(-6): 비치명, 주행 무관.
