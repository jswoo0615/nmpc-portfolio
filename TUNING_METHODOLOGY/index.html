    ControlCore 파라미터 튜닝 방법론 (2026-07-19)

    ROS2 nmpc_ros2::ControlCore(Jetson 배포 경로)의 코너 진입 횡오차를 줄이기 위한 결정적 파라미터 튜닝의 전 과정 기록. 핵심 교훈: 재현 불가능한 환경에서는 튜닝하지 마라.

    1. 문제: 튜닝이 불가능한 상태

    • 증상: HIL/브리지 주행에서 기준 경로 최대 cte가 런마다 0.7~1.4로 크게 출렁. 같은 파라미터도 매번 다른 값 → "q_d 700이 500보다 나은가?"를 판정 불가.
    • 근본 원인: CARLA 비동기 모드 + 제어기 별도 프로세스.
    • 브리지가 CARLA를 async(벽시계 자유진행)로 구동 → 물리 서브스텝 타이밍 비결정적.
    • NMPC 노드가 독립 프로세스 20Hz → 어느 명령이 어느 물리 스텝에 실리는지 비결정적.
    • 중요 인식: 동기 모드로 바꿔도 제어 루프 결합은 여전히 비결정적이다. 동기 모드는 물리만 결정적으로 만들 뿐, 별도 프로세스 제어기의 명령 도착 타이밍은 벽시계 지터가 남는다. 진짜 재현성은 제어기를 물리 tick 루프와 같은 프로세스에 락스텝으로 넣어야 얻어진다 (이것이 원래 Python 파이프라인 nmpc_carla_runner.py가 pybind 인프로세스 + world.tick()으로 재현 가능했던 이유).

    2. 해결: 결정적 락스텝 튜닝 하니스

    • src/control_core_py.cpp: ControlCore를 pybind로 노출 (NMPC_HORIZON=50으로 빌드 — Jetson 타겟과 동일 지평선이라 튜닝 결과가 실기에 그대로 유효).
    • nmpc_ros2/scripts/tune_harness.py: 단일 프로세스에서 상태 읽기 → core.step() → apply → world.tick()을 CARLA 동기 모드로 락스텝.
    • 검증: 동일 파라미터 2회 실행 → ticks/fd_max/fd_avg/avg_v 완전 일치 (901틱, fd_max 0.835, avg_v 30.8). 재현성 확보 = A/B가 비로소 의미를 가짐.
    • 지표 선택: 하니스의 raw-웨이포인트 cte는 코너 꼭지점에서 한 틱 스파이크(아티팩트)가 있어, 솔버 자체 횡오차 |frenet_d|를 1차 지표로 사용.

    3. 진단: 최대 오차의 정체

    기준선 CSV 분석: - max|fd|=0.835는 딱 코너 진입 1회(t≈8.4s), avg|fd|=0.055로 추종 자체는 우수 (fd>0.7은 901틱 중 7틱뿐). - 그 지점에서 조향이 +0.600 rad에 3틱 연속 포화 + 진입 램프가 정확히 0.2 rad/틱(=slew 4.0×dt 0.05)으로 제한 → 두 인위적 상한이 병목임을 특정. - raw cte "0.97"(t=9.0)은 fd가 −0.15에 불과한 측정 아티팩트로 분리.

    4. 튜닝: 순차 단일 파라미터 스윕

    조합 폭발을 피해 한 번에 하나씩 스윕, 각 값은 결정적 하니스로 1회 주행:

    4-1. steer_slew (조향 슬루 한계)

    값 fd_max 판정
    4.0 (기준) 0.835 진입 램프 병목
    6.0 0.752 병목 해소
    8.0 / 12.0 0.752 무변화 → 6에서 슬루는 더 이상 병목 아님

    → 6.0 채택 (병목 제거하는 최소값, 시동 완충 유지). 이후 병목은 조향 크기 상한.

    4-2. u_max_steer (조향 크기 상한) — 반직관적 결과

    값 fd_max avg_v 판정
    0.6 (기준) 0.752 30.8 최적
    0.8 1.276 16.0 과보정→불안정 붕괴
    1.0 0.955 30.7 악화
    1.22 1.36 16.6 붕괴

    → 조향 권한을 더 주면 솔버가 과보정→진동→불안정. 0.6 상한은 실은 안정화 장치. 0.6 유지. 코너 포화는 "더 꺾어서" 풀 문제가 아님.

    4-3. q_kappa_track (곡률 피드포워드) — 비단조/지뢰밭

    값 fd_max avg_v 판정
    200 (기준) 0.752 30.8
    400 0.731 30.8 강건한 개선
    700 0.786 16.8 동결(동심원 호 계곡)
    1000 0.636 30.8 수치 최고이나 700 옆 지뢰밭

    → 1000이 수치상 최고지만 바로 옆 700이 코너 동결. 이는 비볼록 OCP의 알려진 국소최적점(동심원 호 계곡, 본편 D1 참조) 지형. 맵 비종속·재튜닝 금지 제약에서 지뢰밭 인접값은 다른 코너에서 동결 위험 → 400 채택(단조 안전 영역).

    5. 결과 및 한계

    • 확정 기본값: steer_slew=6.0, q_kappa_track=400 (control_core.hpp 구조체 기본값).
    • 성과: 코너 진입 max|fd| 0.835 → 0.731 (−12%), avg|fd| 0.055 유지, avg_v 30.8 유지.
    • 정직한 한계: avg|fd|는 이미 우수하고, 유일한 흠인 코너 진입 0.73은 가중치 튜닝으로는 여기까지. 더 줄이려면 가중치가 아니라 구조 변경이 필요: (a) 예측 지평선 연장(H↑ — Jetson 실시간성과 트레이드오프), (b) 코너 프리뷰(진입 전 조향 램프 예견). → 다음 개선 사이클.

    6. 재사용 가능한 튜닝 원칙

    1. 재현성이 먼저다. 결정적 하니스 없이는 어떤 스윕도 노이즈를 쫓는 것.
    2. 올바른 지표. raw cte의 아티팩트를 solver frenet_d와 분리.
    3. 순차 단일 파라미터. 그리드 대신 한 번에 하나, 병목→다음 병목 순.
    4. 비단조를 경계하라. 수치 최고값이 불안정 basin 옆이면 배제 — 강건성 > 최적값.
    5. 반직관 결과를 존중하라. u_max를 올리면 나빠진다는 데이터를 받아들일 것.
