    전체 통합 아키텍처 (2026-07-19 기준)

    자체 구현 NMPC를 Autoware Universe 전체 스택 + CARLA에 통합해 실주행까지 검증한 구성. 아래 다이어그램은 실제 토픽/파일명 기준이며, 축소 구성(perception:=false)에서 우리가 채워 넣은 브리지들이 어디에 왜 들어가는지까지 표시한다.

    최적화 문제 정의

    항목 내용
    상태 (8) (s, d, μ, v_x, v_y, r, α_f, α_r) — Frenet + 차체속도 + 요레이트 + 전/후륜 슬립각
    입력 (2) (δ, a) 조향각 · 종가속도
    Horizon NMPC_HORIZON = 500 (데스크톱) / 50 (Jetson, 컴파일 정의), dt = 0.05
    비용 Q_d, Q_μ, Q_Vx, Q_Vy, Q_r, Q_αf=Q_αr=200, Q_κtrack + R_steer, R_accel, R_steerRate, R_accelRate
    Hard constraint 횡방향 회랑 d ∈ [d_min, d_max] (적응형), 조향 \|δ\| ≤ 0.6 rad
    속도 hard constraint 아님 — 곡률 기반으로 레퍼런스 target_vx를 낮춰 반영

    3. 속도 정책 — 제동거리 인지형 곡률 선제감속

    flowchart LR
    A["Autoware 궤적<br/>per-point velocity"] -->|"목표 근처만<br/>(goal_zone_m=12m)"| D["target_vx"]
    B["cruise_speed_mps<br/>= 8.33 (30km/h)"] -->|"중간 구간"| C
    C["곡률 캡<br/>look_m = max(20, v²/2a_brake)<br/>v_curve = √(a_lat_max/κ)<br/>v_allow = min(√(v_curve² + 2·a_brake·d))"] --> D
    D --> E["ControlCore → IPM"]

    style C fill:#fff5b1

    왜 Autoware 속도를 중간 구간에서 무시하는가 — Town10HD의 코너는 대부분 교차로라 behavior_velocity_planner가 (perception이 꺼져 안전 확인 불가) ~5km/h 기어가기를 넣는다. 그래서 순항 속도 주도권을 컨트롤러로 회수하고, 코너 감속은 a_lat_max가 담당한다. 단, 이로 인해 맵 speed_limit이 제어기 상한으로 전달되지 않는 공백이 생겼다 (개선 예정).

    ⚠️ 오늘 생긴 실제 안전 공백: 컨트롤러를 unload하면서 끊긴 control_state 진단을 control_state_diag.py로 브리지했는데, level을 OK로 하드코딩했다. 즉 NMPC가 fallback 상태여도 Autoware는 정상으로 인식하고 AUTONOMOUS를 유지한다. → SolverStatus / fallback_triggered를 실제 진단 레벨(OK/WARN/ERROR)로 매핑해야 한다.

    5. 두 종류의 안전은 층이 다르다

    능동 안전 (Active) 수동/축퇴 안전 (Fallback)
    트리거 장애물 감지 시스템 고장
    주체 Planner / NMPC 제약 Safety Supervisor
    동작 회피 궤적 생성 위험 최소화 (정지/갓길)
    우리 구현 회랑 제약, 곡률 감속 SolverStatus + SafeBuffer + Fallback
    Autoware behavior_velocity, AEB MRM Handler

    제어기 내부 fail-safe("내 솔버가 실패해도 차를 망가뜨리지 않는 입력")와 차량 시스템 MRM("자율주행을 계속해도 되는 상태인가")은 범위가 다르며, 둘 다 필요하다. 우리는 ①이 잘 되어 있고 ②로 가는 보고 경로가 비어 있는 상태다.

    6. 실행 절차 요약

    sequenceDiagram
    participant U as 사용자
    participant C as CARLA
    participant A as Autoware e2e
    participant N as 우리 노드
    U->>C: CarlaUE4.sh (RPC 2000 대기)
    U->>A: carla_full (nohup, python3.10 강제)
    A->>C: 연결 · ego 스폰 · sync tick
    A-->>U: /clock + /localization/kinematic_state
    U->>A: stop_controller (controller_node_exe unload)
    U->>N: support + nmpc (nohup)
    N-->>A: control_cmd 단독 발행 확인
    U->>A: find_routable_goal_heading → set_route
    U->>A: change_to_autonomous
    A->>C: 주행
